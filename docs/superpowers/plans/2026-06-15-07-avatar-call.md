# 3D Avatar Bot Call Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When the call participant is the ugly-bot, render a real 3D avatar that lip-syncs to the bot's TTS (visemes) and shows the bot's speech as captions in the existing transcript, instead of the static emoji tile.

**Architecture:** Reuse the framework's `TalkingAvatar` (`ugly-app/three/client`) — it takes a GLB `src` + an `AnalyserNode` from `useTTS().analyser` and animates the mouth. We host the ugly-bot GLB in R2 (blob.ugly.bot has no CORS — re-host per platform note). When a bot turn arrives, the client plays it through `useTTS` with `{ onViseme, onSubtitleIndex }`; visemes drive the avatar, the subtitle index reveals the caption (already wired into the transcript from Plan 06). In `VideoCall`, the bot tile renders `<BotAvatarTile>` instead of the emoji when `participant.isBot`.

**Tech Stack:** React, `ugly-app/three/client` (`TalkingAvatar`), `ugly-app/client` `useTTS`, three.js (already a dep), R2 blob hosting, vitest.

> Verified facts: `TalkingAvatar` props `{ src, analyser, speaking, framing, background, onReady }`; `useTTS(socket)` returns `{ play, analyser, playing, currentWord }` and `play(text,{onViseme,onSubtitleIndex})` streams visemes/word-index; `AvatarModel.showViseme(...)` is the lower-level path if needed. `three` is installed but unused in ugly-chat. `VideoCall.tsx` renders bots as a static emoji tile today.
> Visual source: `mockups/call-bot.html` (3D head + HUD + captions).

### Task 2: BotAvatarTile component

**Files:**
- Create: `client/components/BotAvatarTile.tsx`

- [ ] **Step 1:** Wrap `TalkingAvatar` with the call HUD chrome from `mockups/call-bot.html` (live model/latency/cost overlay reuses the receipt classes; orange rim handled by the GLB lighting/scene). Drive it from `useTTS`:

```tsx
// client/components/BotAvatarTile.tsx
import React, { useEffect } from 'react';
import { TalkingAvatar } from 'ugly-app/three/client';
import { useTTS } from 'ugly-app/client';
import { BOT_AVATAR_URL, ttsVoiceForBot } from '../lib/avatar';

export function BotAvatarTile({ socket, botId, speakText, onSubtitleIndex }:
  { socket: any; botId: string; speakText: string | null; onSubtitleIndex?: (i: number) => void }): React.ReactElement {
  const tts = useTTS(socket);
  useEffect(() => {
    if (!speakText) return;
    void tts.play(speakText, { voice: ttsVoiceForBot(botId), onSubtitleIndex });
  }, [speakText]); // play each new bot turn
  return (
    <div style={{ position: 'relative', width: '100%', height: '100%', background: '#07080B' }}>
      <TalkingAvatar src={BOT_AVATAR_URL} analyser={tts.analyser} speaking={tts.playing} framing="head" />
    </div>
  );
}
```

- [ ] **Step 2:** Must call `tts.warmup()` inside the user gesture that starts/joins the call (Safari AudioContext unlock) — do this in the call-join handler in `VideoCall`/`CallLayout`.

- [ ] **Step 3: Verify (build).** `npm run build` — three.js + TalkingAvatar import resolve. Note: the framework warns server-mode dynamic imports must be stubbed; this tile is client-only (rendered in the browser) so it's fine.
- [ ] **Step 4: Commit** — `git commit -m "feat(avatar): BotAvatarTile (TalkingAvatar + TTS)"`
