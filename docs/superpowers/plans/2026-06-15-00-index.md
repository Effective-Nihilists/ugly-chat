## Shared conventions (apply to every plan)

- **Repo:** `/Users/admin/Documents/GitHub/ugly-chat`. Framework: `ugly-app` (Cloudflare Workers).
- **Tests:** `vitest`, config at `vitest.config.ts`, specs in `tests/unit/**/*.test.{ts,tsx}`, env `node`. Coverage gate (90%) applies only to `shared/**` and `server/**` — so **all testable logic goes in `shared/` or `server/`**, never buried in a `.tsx`. Run all: `npm test`. Run one file: `npx vitest run tests/unit/<file>.test.ts`.
- **Client-only visual changes** (CSS/JSX) are not unit-testable under the node env. For those, the verification step is: `npm run dev`, open the relevant route, and compare against the matching `mockups/*.html` frame. Where a `.test.tsx` is warranted (component logic), add `environment: 'jsdom'` via a top-of-file `// @vitest-environment jsdom` comment and install `@testing-library/react` (the plan that first needs it sets it up).
- **Visual source of truth:** the `mockups/*.html` files contain the exact target markup, class names, and CSS. Plans reference them by file rather than re-pasting hundreds of lines (DRY). Port the styles into `client/styles.css` (global) and component JSX.
- **Brand invariants** (from `app/youtube/BRAND_GUIDELINES.md`): square corners on chrome, 1px hairlines, one accent (`#FF5500`, green in Vim), Plus Jakarta Sans + JetBrains Mono + Inter, receipts on screen, no emoji (lucide icons only).
- **Commits:** one per task, conventional-commit style, end with the Co-Authored-By trailer.
- **Branch:** create a feature branch per plan (e.g. `feat/theme-system`); do not commit to `main`.

## Execution

Each plan ends with the standard handoff. Recommended: **subagent-driven-development** (fresh subagent per task, review between). Start with Plan 01.
